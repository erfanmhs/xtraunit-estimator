# XtraUnit Scope Taxonomy (CSI MasterFormat — draft v2)

The canonical **subcategories** the AI uses when drafting a scope of work.
Standard: formal CSI MasterFormat section titles + a short scope note — the
wording professional GCs and public-works/government bids use. Work-package
level (how you buy/price it), grouped by CSI division.

**Rules for the AI:**
- Every scope line maps to ONE subcategory and uses its **standard label**; the
  project-specific spec/location is appended after an em-dash.
- Granular work **rolls up** into its package; include a subcategory **only if
  the project has that scope**.
- **The AI MAY add a new subcategory** under the correct division when the
  project has scope that genuinely fits none listed — but only at this same
  work-package level and in the same formal CSI-style wording (section title +
  short scope). It must not invent redundant near-duplicates of existing ones.
- Labels double as the **cost-catalog vocabulary** (scope → pricing → history all
  match). `(c)` = conditional (only if present). `(o)` = usually owner's-cost.
- Later: this list becomes user-editable (add / rename / remove per company).
